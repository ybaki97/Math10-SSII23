üéâ Final Project Instructions#

üéØ Goal#

The purpose of the final project is to submit a Deepnote project which analyzes a dataset of your choice. This type of project could look great in a portfolio when applying to internships and jobs.

üìä Logistics#

The following are required to pass the final project.

  • Due at 11:59pm PDT on Wednesday, September 13th, 2023. No late submissions will be accepted, so be sure to leave plenty of time to submit the link to your project on Canvas. Feel free to upload the link ahead of time so that you don‚Äôt need to worry about it.

  • Unlike our homeworks, this is an individual assignment. Everyone will need to submit their own original work.

  • Use the Project Template available in the Project folder on Deepnote.

  • Using the Share button at the top right on Deepnote, enable public sharing, and enable Comment privileges. Then submit the created link on Canvas.

  • The primary focus of the project must be on something involving data, and primarily using one or more datasets that weren‚Äôt covered in Math 10. You can use datasets that are built in to a Python library like Seaborn or scikit-learn, or you can upload the dataset to Deepnote yourself, such as a dataset that you downloaded from Kaggle, that you got from openml.org, or that you got from UC Irvine‚Äôs own Machine Learning Repository. (Warning. Deepnote does not allow datasets larger than 100mb to be uploaded.)

  • The project should clearly build on the Math 10 material. If you‚Äôre an expert in Python material that was not covered in Math 10, you are of course welcome to use that, but the project should use the topics of Math 10 in an essential way (see the rubric below).

  • Answer the questions at the top of the Deepnote template, especially the question regarding whether you want your project posted in the course notes. Here are examples of student projects from Spring 2023, examples of student projects from Fall 2022, and examples of student projects from Spring 2022.

  • Anything that is taken from another source (either the idea for the project or a piece of code, even if you edit that code) should be explicitly referenced with a link. (For example, you could write, ‚ÄúThe configuration of this Altair chart was adapted from [link].‚Äù).

üíØ Rubric#

The following components are required to receive a passing mark on the final project.

  1. Clarity Is the project clear and well-organized? Does it use good coding style? Is the code Pythonic (for example, avoiding unnecessary for loops) and DRY (avoiding unnecessary repetition)? Does it explore one or more clearly described datasets, and is it clear where those datasets came from? Use text and markdown throughout the project to help the reader understand what is going on. Is the reasoning clear? (It‚Äôs fine if you have negative results like, ‚Äúso there is no clear connection between these variables‚Äù. In fact, I prefer those sorts of results as opposed to unbelievable claims.) Give your project a relevant title (like ‚ÄúSpecies of penguins‚Äù rather than ‚ÄúMath 10 project‚Äù).

  2. Machine Learning Does the project explore the data using a variety of tools from scikit-learn? Does the project refer to essential aspects of data analysis, such as over-fitting, or the importance of a test set, or feature engineering, or the difference between classification and regression?

  3. pandas Does the project make essential use of pandas to explore the data? Using pandas to read and store the data is not sufficient. It should also be used either to clean the data, or to analyze the data, or for performing feature engineering.

  4. Altair Does the project include a variety of interesting charts, made using Altair? Do we learn something about the data from these charts?

  5. Extra Does the project include material that was not covered in Math 10? (This could include different libraries, different machine learning algorithms, or deeper use of the libraries we covered in Math 10. Some possible topics are included in the Possible Extra Topics notebook, but you‚Äôre very encouraged to choose your own.)

üíÅ‚Äç‚ôÄÔ∏è Advice#

Below you will find some advice about the project.

  • Don‚Äôt repeat the same technique multiple times unless it‚Äôs really essential. For example, having one interesting chart and a different less interesting chart is better than having the same interesting chart made for two different datasets. (Of course, if you can find a DRY way to make many related charts, then that‚Äôs great.)

  • It‚Äôs easy to spend lots of time trying to find the perfect dataset, but I‚Äôd advise against letting this take up too much time. I‚Äôd rather you do an interesting analysis of a boring dataset than a less interesting analysis of a super cool dataset.

  • Don‚Äôt make unreasonable claims. I‚Äôd much rather you say ‚Äúwe didn‚Äôt see any relationship bewteen A and B‚Äù than try to force a claim that just doesn‚Äôt seem true.

  • The main point of this project is to show me what you have learned in Math 10 this summer session. If you are already very advanced in python and write your project on topics that weren‚Äôt covered in our class, that will not necessarily receive a passing grade.

  • Include many markdown cells (possibly very short, just one sentence) explaining what you are doing. (These are different from Python comments.) If you want to use bold text in your markdown cell, surround the text by two underscores __like this__. Be sure you‚Äôre doing this in a markdown cell, as opposed to in a Python comment after a # symbol.

  • This is already written above, but it‚Äôs worth repeating. Please err on the side of referencing everything. ‚ùì Frequently Asked Questions#

Here you will find some answers to frequently asked questions. I‚Äôll also update this section as more questions come in.

Q: Is there a length requirement? A: There is no specific length requirement, but as a rough estimate, spending approximately 12 productive hours on the project would be a good amount. (The word ‚Äúproductive‚Äù is important. Time spent browsing tutorials or datasets is not productive in this sense if you don‚Äôt end up using that material.)

Q: What should I focus on? A: The primary content should be from one or more of the Math 10 tools (pandas, Altair, scikit-learn). If you like pandas much more than Altair, for example, then it‚Äôs okay to go more in-depth in the use of pandas, and less in-depth in the use of Altair, but you still need to include both.

Q: Do I need to get original research results? A: No! If you explore the data in an interesting way, but can‚Äôt find any interesting conclusions, that‚Äôs fine. In fact, I prefer that to making claims that the data does not support.

Q: What if a lot of my work was done cleaning the data, but it does not appear in the project? A: Include those cleaning steps in the project! (If the work was done outside of Python, like in Excel, then that portion will not count.)

Q: Can I use a different plotting library? A: You need to use Altair for the Altair portion of the rubric. You can definitely use a different library (like Seaborn, Plotly, or deeper aspects of Matplotlib) for the extra part of the rubric.

Q: What if I‚Äôm worried my project is too short? A: It‚Äôs fine to switch to a different topic halfway through your project. If you finish what you originally planned and want to start something different, that‚Äôs fine, even if it uses a different dataset. This isn‚Äôt like a history paper where you should have a coherent focus from start to end.

Q: Can you look at my project early? A: I‚Äôm happy to give a quick first impression and let you know if I have any immediate suggestions. (I won‚Äôt have time to read it thoroughly until the final submission.)

Q: Do I need to post my project in the course notes? A: No, that is totally optional, but it might be a nice idea if you would like to show the project if applying for an internship, job, or grad school, for example.

Q: How can I use an Excel file instead of a CSV file? A: If you have an Excel file (with an extension .xlsx or .xls) instead of a csv file, you can try using pd.read_excel instead of pd.read_csv. That usually doesn‚Äôt work for me, but if I first try !pip install openpyxl and then try to use pd.read_excel, it usually works. It might be easier to just open the file in Excel, save it as a comma-separated csv file there, and then upload that csv file. Of course another option is just to open the Excel file in Excel or Google Sheets and then save it as a csv file from there.
